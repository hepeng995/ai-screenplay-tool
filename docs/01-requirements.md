# 项目需求文档 (PRD)

## 1. 项目背景

很多小说作者希望将自己的作品改编成剧本，但手动改编门槛高、效率低。本工具利用 AI 大模型自动将小说文本转换为结构化 YAML 剧本，降低改编门槛，提升效率。

**竞赛题目**：七牛云校园编程大赛 — AI 辅助剧本创作工具

## 2. 目标用户

| 用户群 | 场景 | 痛点 |
|--------|------|------|
| 小说改编编剧 | 批量章节改编 | 手动格式化耗时 |
| 影视专业学生 | 学习剧本结构 | 缺少结构化参考 |
| 内容创作者 | 小说→短剧脚本 | 改编门槛高 |

## 3. 核心功能（Must Have）

| # | 功能 | 输入 | 输出 | 验收标准 |
|---|------|------|------|---------|
| F1 | 文件上传 | .txt/.md 文件 | 文件信息 | 支持≤10MB，拖拽上传 |
| F2 | 章节切分 | 纯文本 | 章节数组 | 自动识别中英文标记，≥3章检测 |
| F3 | AI 转换 | 章节文本 | YAML 剧本 | mimo-v2.5，Edge Runtime，≤3并发 |
| F4 | YAML 预览 | YAML 文本 | 树形结构 | 幕/场/台词三级展示 |
| F5 | 在线编辑 | YAML 文本 | 实时校验 | Zod Schema 校验，错误定位 |
| F6 | Schema 校验 | YAML 文本 | 校验结果 | 前后端双重校验 |
| F7 | 七牛云上传 | YAML 内容 | 云端文件 | Token 鉴权 + 客户端直传 |
| F8 | 七牛云下载 | 文件 Key | YAML 内容 | 签名 URL 下载 |
| F9 | 文件导出 | YAML 内容 | 下载文件 | YAML / JSON 双格式 |

## 4. 非功能需求

| 类别 | 要求 | 实现 |
|------|------|------|
| 性能 | AI 单章响应 ≤90s | Edge Runtime 25s + 章节切分 |
| 安全 | API Key 不泄露 | 服务端环境变量，SK 仅 Edge API |
| 可用性 | 零成本部署 | Vercel Hobby + 七牛云免费额度 |
| 容量 | 单文件 ≤10MB | 客户端 + PutPolicy 双重限制 |
| 并发 | AI 并发 ≤3 | withConcurrencyLimit 信号量 |

## 5. 约束（Must NOT Have）

- ❌ 数据库 / 用户登录 / 富文本编辑器
- ❌ AI 流式响应 / 版本历史 / 多模板 / i18n
- ❌ PDF/DOCX 导出（仅 YAML + JSON）
